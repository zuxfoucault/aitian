# Historical tier — plans

Per-milestone implementation plans. **Allowed to go stale.** A record of the plan at the time, **not**
the source of truth — the maintained `docs/*.md` and the code are.

- Don't cite these where they conflict with the code or the root `CLAUDE.md`.
- **Migrate before you retire:** lift any still-current decision into a maintained doc before the plan
  goes stale.
- **Kept, never deleted.** The files stay forever as decision history.

Add new milestone plan files alongside this README (e.g. `2026-07-mvp.md`); leave the old ones in place.
